# Adapter: JSON Only

This is the simplest possible version.

## Backlog

Use a local JSON file such as `backlog.json`.

Suggested structure:

- key
- title
- type
- status
- priority
- labels

## Published Context

Use markdown files in the repo:

- `docs/reports/YYYY-MM-DD-status.md`
- `docs/decisions/`

## Notes

- Best for solo projects and experiments
- Good enough until the project needs a real shared system
