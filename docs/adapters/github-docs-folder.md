# Adapter: GitHub Issues + Docs Folder

This is a strong low-friction setup if your project already lives mostly in GitHub.

## Backlog

Use GitHub Issues for:

- bugs
- enhancements
- in-progress work tracked through issues and PRs

## Published Context

Use a repo `docs/` folder for:

- weekly reports
- decisions
- release notes

Suggested path:

- `docs/reports/YYYY-MM-DD-status.md`

## Notes

- This setup is easier to share publicly than Jira/Confluence.
- It also keeps the workflow close to the code.
